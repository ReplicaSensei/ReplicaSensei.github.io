三大修饰符:abstract static final

Posted on 2020-07-28

三大修饰符

1.abstract

特点：

  • 修饰类:不能new对象，但可以声明引用。

  • 修饰方法:只有方法声明，没有方法实现。（需包含在抽象类中）

  • 抽象类中不一定有抽象方法，但有抽象方法的类一定是抽象类。

  • 子类继承抽象类后，必须重写父类中所有的抽象方法，否则子类还是抽象类。

2.static

特点:

  • 静态方法中可以直接访问静态成员，不可以直接访问非静态成员，不能使用this和super关键字。
  • 非静态方法中可以直接访问静态成员。
  • 静态方法可以继承，不能重写、没有多态。

    abstract 不能和static同时使用。

3.final

特点:

final修饰类：此类不能被继承

final修饰方法:此方法不能被覆盖

final修饰变量:

1.final修饰局部变量(常量)赋值方式：
  • 声明时赋值
  • 只能赋值一次
2.final修饰实例变量(常量)赋值方式：
  • 声明时赋值
  • 构造方法中赋值
  • 只能赋值一次
3.final修饰类变量(常量)赋值方式：
  • 声明时赋值
  • 在静态代码块赋值
  • 只能赋值一次
4.final修饰对象常量：
  • 对象地址不可修改
  • 可修改对象属性

abstract不能和final同时使用
